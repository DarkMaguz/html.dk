Skjul udvalgte CSS-koder for Netscape 4

af Jonas Astrup, HTML.dk
Sidst opdateret 22. januar 2001

Alle som bruger Cascading StyleSheets (CSS) kender til problemerne med Netscape Navigator 4. Denne artikel handler om en metode, som kan være en stor hjælp når man benytter CSS - og gerne vil tage hensyn til brugere som ikke har fået opgraderet deres browser.

Bemærk i den forbindelse at Netscape har udsendt version 6 af deres browser. Netscape 6 understøtter både HTML og CSS standarderne. Artiklen her handler udelukkende om den gamle version 4 - som der stadig er en vis procentdel af brugerne, som benytter.

Metoden benytter sig udelukkende af W3C-standarder, og bruger ikke scripting som f.eks. javascript eller lignende

To typer problemer

Netscape 4 indeholder mange fejl i forbindelse med understøttelsen af CSS. Man kan opdele manglerne i to kategorier af problemer:

1. CSS funktionaliteter som slet ikke understøttes af Netscape 4
Her er der som regel ikke så meget at gøre - og ofte må man blot acceptere at Netscape 4 ikke kan vise siderne helt så avanceret som nutidige browsere.
2. CSS funktionaliteter som delvist eller fejlagtigt understøttes
Denne type fejl er som regel langt værre. Konsekvenserne kan være totalt uberegnelige - siderne kan være helt ubrugelige eller få brugerens PC eller browser til at bryde sammen.

Det er de sidstnævnte fejl man kan komme udenom ved at skjule netop de problematiske dele af CSS-koden for Netscape 4. Lad os kigge på hvordan man gør.

@media reglen

Tricket er at bruge @media reglen. Denne regel giver dig mulighed for at specificere forskellige CSS koder til forskellige medier. Det vil sige at man f.eks. kan få den samme side til at fremtræde forskelligt på skærm og print. Se eventuelt artiklen Lav dine sider printervenlige med CSS for en demonstration af hvordan det kan lade sig gøre med to forskellige stylesheets.

Med @media reglen kan man i det samme stylesheet angive hvilke medier dele af koderne skal gælde for.

Et eksempel kunne være at overskrifter (<h1>) skal være røde på en skærm, men blå på en udskrift:

@media screen {
h1 {color:red;}
}

@media print {
h1 {color:blue;}
}

Systemet giver sig selv ved at betragte ovenstående - men netop denne funktionalitet understøttes ikke af Netscape 4 - hvilket vi kan udnytte til at gemme nogle CSS koder.

Sådan gemmer du koder for Netscape 4

Lad os sige at du har kodet noget snedigt CSS, som f.eks. floater et formular-felt lige som du vil have det. Men desværre får dit stylesheet Netscape 4 til at bryde helt sammen.

Istedet for at opgive dine smarte koder helt, kan du vælge at gemme de problemskabende koder for Netscape 4 ved at skrive dem specielt til skærm med @media reglen - sådan her:

@media screen {
#form2 {float:right;}
}

Dette resulterer i at alle andre browsere end Netscape 4 forstår CSS-koden. Netscape 4 ignorerer derimod fuldstændigt alt hvad der står angivet til @media screen - og giver dermed ikke problemer. Hvis du gerne vil lave en alternativ løsning for Netscape 4 kan du læse mere nedenfor.

Sådan laver du en speciel løsning for Netscape 4

Man kan på samme måde lave en speciel løsning kun til Netscape 4. Ideen er at man først angiver den CSS-kode som skal gælde for Netscape 4. Derefter ændrer man på egenskaberne for det samme element - men denne gang skjult for Netscape 4 bag @media reglen.

Lad os kigge på et simpelt eksempel hvor overskrifter (<h1>) vises med fonten arial i Netscape 4 - men med Times New Roman i alle andre browsere:

h1 {font-family: arial;}

@media screen {
h1 {font-family: Times New Roman;}
}

Denne effekt opnås altså fordi Netscape 4 kun forstår det som står udenfor @media reglen - mens alle andre browsere læser ændringen af fonten længere nede i stylesheetet - og dermed lader det sidst specificerede tage effekt.

Held og lykke med koderne.
